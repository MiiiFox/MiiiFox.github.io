hello hexo

发表于 2019-12-26 更新于 2019-12-30 分类于 能动的认识世界和改造世界

hexo+github搭一个个人博客。

必要环境

git：git 默认安装

node.js：node.js 默认安装

hexo安装

hexo官网

在非系统盘任意方便位置新建文件夹blog,在文件夹内右键选择 git bash there 进入git终端。

git终端内键入 npm install hexo-cli -g 回车确认全局安装hexo，然后键入 hexo init 将hexo安装到当前路径下（由于已经在文件夹blog内打开终端执行，所以hexo安装在了blog文件夹内，若不是在blog内打开终端，请cd到blog文件夹下再执行 hexo init )

等待一会后安装成功，blog下会出现一堆文件,大概如下图所示。

themes文件夹里装了一些主题，默认是landscape；source文件夹下存放资源，比如写的博客文件，博客的分类、关于、归档、标签页面等。_config.yml是整个网站的基础配置，在themes里的主题中也有同样的一个文件，那个是主题的配置。

在git终端内运行 hexo g ，该命令根据当前的配置和资源信息生成完整的web文件，可看到多了一个public文件夹，该文件夹内包含web网站运行的所有文件。运行 hexo s 在本地开启web服务，本地web服务将以public为根目录展示网站，浏览器访问 localhost:4000 即可看到博客网站的初貌。

hexo设置

hexo主题

hexo主题有很多开源主题，可选一个喜欢的到其github下载zip包并解压，然后复制到blog下的themes里，或执行git clone https://github.com/theme-next/hexo-theme-next themes/next(以主题next为例)，成功后打开blog下的_config.yml，将 theme: (注意冒号后面又一个空格)字段后的文字改为next（或存放其他主题的文件夹名）即可。运行 hexo g 和 hexo s 查看是否配置正确。

hexo上传github

注册github，新建一个库，命名为xxxx.github.io ，xxxx为你的github用户名，如果用户名为oreo就命名为oreo.github.io。点击右上角头像，点击setting，在左侧Personal settings栏里找到’ssh and gpg keys’选项,新建一个ssh key，“title”栏里随意填写内容。

在blog下打开git终端，运行 ssh-keygen -t rsa -C "xxxxx@qq.com" ,其中引号内填写自己注册github时填写的邮箱地址。此步生成ssh，在系统盘中找到.ssh文件夹（一般在C:\Users\用户名\ 里），其下存放ssh密钥。打开以.pub结尾的文件，将其中的内容全部复制到刚刚新建的ssh key的“key”栏中。点击“add ssh key”保存。

继续在git终端里运行 npm install hexo-deployer-git --save 安装上传插件，继续执行 hexo clean（该命令将public文件夹删除，如果后期遇到网页显示错误、主题不生效等问题时可先运行该命令清除网站内容，再重新生成并上传网站文件）、 hexo g 、 hexo d 即可将网站部署到github，浏览器打开 http://xxxxoreo.github.io 就可预览了。

绑定个人域名

阿里云、腾讯云注册一个域名，有钱com没钱top。

去github里xxxxoreo.github.io库的右侧的设置下，找到github page这一栏，把注册的域名填上，保存。

打开电脑终端（cmd），执行 ping xxxxoreo.github.io 得到该库的ip地址。

打开注册域名的域名解析服务，添加两条记录，形式如下所示：

CNAME记录类型即该网站域名的别名域名（可理解为人的外号），记录值就填xxxxoreo.github.io，这样你注册的域名和这个域名都可以代表你的网站。A记录类型即网站主域名映射的ip地址记录，其记录值填刚刚ping得到的ip值，这样无论访问哪个域名，都能正确解析到ip地址对应主机的博客网站。

最后，在blog下的source文件夹中新建一个名为CNAME的文件（注意没有任何后缀）,里面写上注册的域名，保存。git终端里依次执行 hexo clean、hexo g 、hexo d，就配置好了。

这样访问注册的域名就能看到搭建的博客了。
